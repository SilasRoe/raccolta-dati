EINGABE: Du bekommst als INPUT eine einzige Zeichenkette mit dem vollständigen Markdown-Inhalt einer PDF (PDF wurde in Markdown umgewandelt).

AUFGABE: Extrahiere alle relevanten Informationen einer Rechnung aus dem Markdown und liefere exakt ein einziges gültiges JSON-Objekt als einzige Ausgabe. Keine Erklärungen, keine zusätzlichen Texte, nur JSON.

ERWARTETES JSON-SCHEMA:
{
  "produkte": [                         // Array mit gelieferten/abgerechneten Positionen (kann leer sein)
    {
      "produkt": string | null,
      "gelieferteMenge": number | null  // Zahl (kein Text), ohne Einheit
    }
  ],
  "nummerRechnung": string | null
}

FORMAT- UND NORMALISIERUNGSREGELN:
- Zahlen: JSON-Zahlen, Dezimalpunkt, keine Tausendertrennzeichen.
- Produkte: wenn keine Produkte erkennbar, "produkte": [].
- Felder, nicht gefunden => null.
- Wenn mehrere mögliche Rechnungsnummern/Daten vorhanden sind, wähle diejenige, die klar als "Fattura", "Invoice", "Rechnung" o. ä. gekennzeichnet ist.
- Ausgabe darf nur die Felder aus diesem Schema enthalten — keine zusätzlichen Felder.

BEISPIEL-AUSGABE:
{
  "produkte":[
    {
      "produkt":"Produkt A",
      "gelieferteMenge":1000
    }
  ],
  "nummerRechnung": "CT123412"
}

WICHTIG: Rückgabe = ausschließlich ein JSON-Objekt, perfekt parsebar. Keine sonstigen Ausgaben.

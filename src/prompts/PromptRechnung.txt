EINGABE: Du bekommst als INPUT eine einzige Zeichenkette mit dem vollständigen HTML-Inhalt einer PDF (PDF wurde in HTML umgewandelt). Optional kann zusätzlich "fileName" oder "filePath" übergeben werden.

AUFGABE: Extrahiere alle relevanten Informationen einer Rechnung aus dem HTML und liefere exakt ein einziges gültiges JSON-Objekt als einzige Ausgabe. Keine Erklärungen, keine zusätzlichen Texte, nur JSON.

ERWARTETES JSON-SCHEMA:
{
  "pdfName": string | null,           // Dateiname, falls vorhanden
  "fullPath": string | null,          // optionaler Pfad
  "datumRechnung": string | null,     // Rechnungsdatum im ISO-Format "YYYY-MM-DD"
  "nummerRechnung": string | null,    // Rechnungsnummer (als String)
  "rechnungssteller": string | null,  // wer die Rechnung ausstellt / Casa Rapp. / Lieferant (falls vorhanden)
  "rechnungsEmpfaenger": string | null,// Rechnungsempfänger / Kunde (falls vorhanden)
  "produkte": [                       // Array mit gelieferten/abgerechneten Positionen (kann leer sein)
    {
      "produkt": string | null,
      "gelieferteMenge": number | null,  // Zahl (kein Text), Einheit wenn möglich getrennt
      "mengeEinheit": string | null,     // z. B. "kg", "pz"
      "preisEinheit": number | null,     // Einzelpreis (ohne Währungssymbol)
      "waehrung": string | null,         // Währung als ISO-Code oder Symbol
      "gesamtpreis": number | null       // Gesamtpreis für die Position
    }
  ],
  "gesamtMenge": number | null         // optional aggregierte gelieferte Menge (wenn klar ersichtlich)
}

FORMAT- UND NORMALISIERUNGSREGELN:
- Datum: ISO YYYY-MM-DD. Fehlend => null.
- Zahlen: JSON-Zahlen, Dezimalpunkt, keine Tausendertrennzeichen.
- Währung: ISO-Code bevorzugt, sonst Symbol.
- Produkte: wenn keine Positionen erkennbar, "produkte": [].
- Felder, nicht gefunden => null.
- Wenn mehrere mögliche Rechnungsnummern/Daten vorhanden sind, wähle diejenige, die klar als "Fattura", "Invoice", "Rechnung" o. ä. gekennzeichnet ist.
- Ausgabe darf nur die Felder aus diesem Schema enthalten — keine zusätzlichen Felder.

EINGABE: Du bekommst als INPUT eine einzige Zeichenkette mit dem vollständigen HTML-Inhalt einer PDF-Datei. Optional kann zusätzlich der Dateiname (z. B. "fileName") oder Pfad (z. B. "filePath") übergeben werden, aber verarbeite primär den HTML-Text.

AUFGABE: Extrahiere alle relevanten Informationen einer Auftragsbescheinigung aus dem HTML und liefere exakt ein einziges gültiges JSON-Objekt als einzige Ausgabe. Keine Erläuterungen, keine Textteile, kein Markdown — nur reines JSON.

ERWARTETES JSON-SCHEMA (Pflichtfelder wenn vorhanden, sonst null / leeres Array):
{
  "datumAuftrag": string | null,     // Datum des Auftrags im ISO-Format "YYYY-MM-DD"
  "nummerAuftrag": string | null,    // Auftragsnummer (als String)
  "kunde": string | null,            // Name/Adresse des Kunden (falls vorhanden)
  "lieferant": string | null,        // Name/Adresse des Lieferanten / Casa Rapp. (falls vorhanden)
  "produkte": [                      // Array: eine Bestellung kann mehrere Produkte/Positionen haben
    {
      "produkt": string | null,      // Produktbezeichnung
      "menge": number | null,        // Menge als Zahl (keine Tausendertrennzeichen, Dezimalpunkt)
      "mengeEinheit": string | null, // Einheit z. B. "kg", "pz", "pcs" (falls im Dokument angegeben)
      "preisEinheit": number | null, // Preis pro Einheit als Zahl (ohne Währungssymbol)
      "waehrung": string | null,     // Währung falls vorhanden (ISO-Code z. B. "EUR" oder Symbol)
      "gesamtpreis": number | null   // Gesamtpreis der Position (wenn angegeben) als Zahl
    }
  ]
}

FORMAT- UND NORMALISIERUNGSREGELN:
- Datum: immer im ISO-Format YYYY-MM-DD. Wenn nur Tag/Monat/Jahr gefunden, normalisiere zu YYYY-MM-DD. Wenn kein Datum ermittelbar, setze null.
- Zahlen: als JSON-Zahlen (z. B. 1234.56). Entferne Tausenderpunkte/commas. Dezimaltrennzeichen = Punkt.
- Währung: gib wenn möglich den ISO-Code (z. B. "EUR"). Wenn nur Symbol vorhanden ist, trage das Symbol ein.
- Produktnamen: bereinige überflüssige Whitespaces, behalte Sonderzeichen.
- Produkte: wenn keine Produkt-Positionen gefunden werden, liefere "produkte": [].
- Felder, die nicht gefunden werden, müssen als null ausgegeben werden (nicht als leeres String).
- Wenn mehrere gleichartige Werte auftreten (z. B. mehrere Daten), wähle dasjenige, das logisch für die Auftrags-Header-Infos ist (oberster Bereich / deutliche Überschrift). Wenn Unsicherheit besteht, nimm den am deutlichsten als Auftragsdatum/nummer markierten Wert.
- Ausgabe darf kein zusätzliches Feld enthalten außer dem oben spezifizierten Schema (zusätzliche meta-Felder sind verboten).
